搭建个人技术博客这件事，往往不是缺少教程，而是缺少一个真正开始的瞬间。

今天这个瞬间来了：我想把自己的安全研究、工程实践、工具折腾和知识系统沉淀下来。第一阶段先不追求复杂主题，也不先把所有自动化都设计完，而是用 Hexo 和 GitHub Pages 跑通一个最小闭环：本地写作、静态生成、自动部署、公开访问。

## 为什么先选 Hexo

Hexo 的好处很直接：

- 文章就是 Markdown 文件，方便后面和 Obsidian 知识库联动。
- 静态站点可以直接托管到 GitHub Pages，成本为零。
- 插件和主题生态成熟，后面可以慢慢换皮肤、加搜索、加统计。
- 构建逻辑足够简单，适合交给自动化工具日常维护。

## 第一阶段目标

第一版只做四件事：

1. 初始化 Hexo 项目。
2. 配好站点基本信息。
3. 用 GitHub Actions 自动部署到 GitHub Pages。
4. 留下一份搭建记录，方便后面复盘成文。

这不是最终形态，只是让写作系统先活起来。

## 后面想接上的东西

Obsidian 联动会是第二阶段重点。理想状态是：平时在 Obsidian 里积累原子笔记、项目记录和文章草稿，需要发布时同步到 Hexo 的 `source/_posts/`，再自动处理 front matter、图片路径、标签和双链。

Hermes Agent 则可以作为第三阶段的日常维护角色。它可以定期检查草稿、提醒可发布选题、跑构建、发现断链，甚至帮我把某些项目日志整理成文章初稿。

先搭台，再唱戏。
